1.1 Problem Background
In today's digital world, when each and every industry is going for digitalization, we were relying on old mess management methods such as mess cards, complaints register, Written Applications etc. So in order to make this management more efficient and transparent. I introduced Umess which is a one stop solution for all these problems which could be an add-on to U-Hostel application. It covers the issue of taking care of hard copies and also provides end-to-end encrypted services with a feature to register anonymous complaints.  

1.2 Objective of the Project
This project aims to build a safe online platform that allows students to use the digital QR code based mess availing feature with providing the feedback to the upper management. This also allows the role based access to the management such as superadmin for Directors, admin for wardens or guards. This is built to give the full access to the hostel management to have a detailed analysis of the food wastage, students reviews, reports for the analysis, adding the global notifications.  

1.3 Proposed Solution / System Overview
For all these problems, Umess comes with a single stop solution which includes the availing the food via QR code system which ensures one student takes the meal once at a time. It also allows a student to see the menu (daily, monthly). Also the students can provide the feedback either anonymously or transparently. It also gives an option to see the notifications instead of going through the multiple notices on the notice boards or whatsapp groups. It also has the feature for faculties to book their venue for their guests (industry visit or any other guests) from a given set of venues. It has Admin dashboard for the complete analysis of the students availed the food, food wastage and lot more.   

1.4 Key Features or Functionality
The system offers secure user sign-up and login, quick verification of ownership, role based access to the admin rights. It has User Management, Menu management, Review management, Notice management, Bookings management, Food wastage management, Reports management. Cloud uploads to provide a scalable solution. Web push notifications. Email Management. QR code based meal availing feature which ensures the one person one meal at a time. It has role based access control for directors, faculties, wardens, guards for each feature. It also involves the cron scheduler which resets the meal availed status on each day. It has an option to download the weekly reports

1.5 Technology / Method Used
The system is built using modern web technologies like the MERN stack (MongoDB, Express.js, React.js, Node.js). Digital QR based profile is integrated through cryptographic hashing methods such as SHA-256 to ensure ownership records are safe and data stays untampered.

1.6 Expected Outcome & Benefits
This will reduce the overhead for the students in managing their mess cards and if they want to give some feedback or reviews anonymously. Also students will be able to directly see the updated menu on the home page itself. They can get an instant notification on their homepage itself. They could also contribute as a volunteer to any occasion that is posted on the page. The admins have full right to get insights of the students and the food quality and quantity. This will also make an  awareness for the students when they see that this much food got wasted and this will be updated on a daily basis. This will also ensure that a notice which is added to this platform will be received to each and every student instead of relying the hard copies of the notices or the Whatsapp groups.  

1.7 Original Contribution Statement
This project presents a one stop solution for modernizing hostel mess management by replacing traditional, manual processes with a secure and scalable system. The proposed platform uniquely combines QR code–based meal authentication, role-based access control, and anonymous feedback mechanisms into a single unified application. Unlike existing fragmented approaches, this system introduces a transparent and data-driven environment that enables real-time monitoring of food consumption, wastage, and student satisfaction.
